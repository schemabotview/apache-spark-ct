What does Spark do to your query before it runs?

Before Spark runs your query, Catalyst builds four plans. Parsed — code as a tree, names unresolved. Analyzed — resolve names against the schema and check types (AnalysisException lives here). Optimized — rule rewrites like predicate pushdown and projection pruning. Physical — pick real operators (broadcast vs sort-merge join); this runs on executors. Parsed + analyzed = correctness; optimized + physical = speed. See all four with .explain(extended=True).
