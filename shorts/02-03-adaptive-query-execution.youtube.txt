How does Spark fix its own query plan?

Old Catalyst optimized upfront, from estimated stats — often wrong. Adaptive Query Execution (AQE) re-optimizes after each shuffle, using the real numbers it just measured: coalesce tiny partitions, switch to a broadcast join, split skewed partitions. Same optimizer, a second chance with facts, not guesses. On by default since Spark 3.2.
